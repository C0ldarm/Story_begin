# SSD (System Specification Document)

## Призначення системи
Система EMS забезпечує централізований збір і аналіз логів та метрик для підвищення доступності та надійності програмних продуктів.

## Основні функції
- FR-01 — Інгестія логів і метрик (підтримка JSON, syslog, Prometheus format).  
- FR-02 — Зберігання та індексація даних з можливістю пошуку за 90 днів.  
- FR-03 — Створення та перегляд інтерактивних дашбордів.  
- FR-04 — Налаштування правил алертів та сповіщень (email, Slack, PagerDuty).  
- FR-05 — Управління користувачами та ролями (RBAC).  
- FR-06 — Генерація аналітичних звітів (PDF/CSV).

## Обмеження та припущення
Припущення: джерела даних підтримують стандартні протоколи (HTTP/gRPC).  
Обмеження: система не зберігає чутливі дані без шифрування; максимальний обсяг одного логу — 10 MB.  
Не підтримує on-premise розгортання без Kubernetes.

## Нефункціональні вимоги
- Продуктивність: обробка ≥ 10 000 подій/секунду, час відповіді API ≤ 200 мс (p95).  
- Доступність: 99.99 % (four nines), multi-AZ розгортання.  
- Масштабованість: горизонтальне масштабування до 100+ нод.  
- Безпека: TLS 1.3, OAuth2/JWT, аудиту логів, GDPR compliance.  
- Надійність: дані не втрачаються при падінні одного сервісу (replication factor 3).
