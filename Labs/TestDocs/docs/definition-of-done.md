# Definition of Done (Documentation)

## Загальні правила
Будь-яка зміна в проекті, що впливає на документацію, вважається завершеною тільки після виконання DoD.

### Типи змін та обов’язкові дії

| Тип зміни                        | Обов’язкові дії з документації                          | Автоматична перевірка          |
|----------------------------------|---------------------------------------------------------|--------------------------------|
| Додавання / зміна API-ендпоінту | Оновити OpenAPI специфікацію (`openapi.yaml`)           | OpenAPI validation + Spectral  |
| Зміна структури запиту/відповіді | Оновити схеми в OpenAPI                                 | Swagger UI build               |
| Додавання нового UI-компонента   | Додати Story в Storybook                                | Storybook build                |
| Зміна існуючого UI-компонента    | Оновити відповідні stories (default, loading, error)    | Storybook build                |
| Зміна архітектури / вимог        | Оновити SSD.md / SDD.md / ISD.md                        | MkDocs build                   |
| Breaking change в API            | Збільшити major версію в openapi.yaml та документації   | Version tag + manual review    |

### Критерії готовності (DoD)
- OpenAPI специфікація валідна та проходить перевірку
- Swagger UI успішно збирається
- Storybook збирається без помилок
- MkDocs сайт генерується без попереджень
- Усі посилання в документації працюють
- Зміни задокументовані в Pull Request описі
- Виконано code review документації
